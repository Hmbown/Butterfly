Butterfly MLX Deployment Note
Date: 2026-04-03

Scope

This path is now framed as a direct comparison between:

- `stock` attention for Qwen 3.5 4B
- `stock` attention for Qwen 3.5 4B with MLX KV quantization
- `butterfly` attention for Qwen 3.5 4B prefill
- `butterfly` attention for Qwen 3.5 4B prefill with stock decode and MLX KV quantization

The public deployment surface should use that language. Legacy names remain compatibility aliases only.

Block-sparse topology note

- MLX also supports `path="block_sparse"` / `--mode block_sparse`.
- That path uses the same staged block-sparse Butterfly topology as CUDA.
- Supported block partner rules are `xor`, `bit_reversal`, `benes`, and `causal_shift`.
- Use `--block-partner-rule causal_shift` for proof-aligned topology experiments.
- The measured 2026-04-03 smoke results in this note are from the older permute-style Butterfly path, so treat block-sparse `causal_shift` as the next experiment until fresh artifacts exist.

The MLX server now provides:

- real OpenAI-compatible SSE streaming
- prompt length and total token caps
- prompt/KV cache reuse with eviction
- optional MLX-LM KV quantization on the full-attention KV cache
- startup warmup
- serial admission control with memory checks
- `/health`
- per-response `butterfly_meta`

Notes:

- In `--mode butterfly`, the server now rejects `--prefill-chunk-size > --query-chunk-size`.
- KV quantization is applied to the working full-attention KV cache after prefill and before stock decode.
- Stored prompt-cache entries remain dense so later prefix extension can still use Butterfly prefill safely.

Takeaway

- The original deploy smoke still shows Butterfly ahead of stock on both prefill and TTFT.
- The surface recheck still shows a clear Butterfly win at `8192`.
- At `2048`, prefill is effectively a wash and TTFT is noisy across reruns. Treat short-prompt results as near-tied.
- The operational claim to carry forward is: Butterfly is the preferred Qwen 3.5 4B prefill path for real deployment-length prompts, and it is clearly better at `8192`.
- End-to-end gains are smaller because decode still leans on the stock path.
- The experimental Butterfly decode path is not ready to enable by default.

Recommendation

4B deployment:

- go for internal deployment only
- deploy `--mode butterfly`
- keep `--butterfly-decode-backend stock`
- keep `--prefill-chunk-size <= --query-chunk-size`
- treat `--kv-bits 4 --kv-group-size 64 --quantized-kv-start 0` as an MLX-native prototype path for decision-grade cache experiments, not a locked production default
- keep serial inference and the current safety caps

9B deployment:

- no-go

Experimental Butterfly decode:

- no-go

Max-context status

- The checkpoint config advertises a theoretical text context ceiling of `262144`.
- The earlier clean Butterfly long-context run in `results/benchmarks/qwen35_4b_mlx/bna_131k_clean` did reach `131072`.
- The fresh guarded no-OOM probes on this active `36 GB` Apple Silicon workstation were safety checks, not clean-room capacity claims.
- Guarded `stock` incremental context reached `21845`, `43690`, and `65536`, then was stopped during the `87381` turn when the memory guard tripped.
- Guarded `butterfly` incremental context reached `21845`, then was stopped during the `43690` turn when the memory guard tripped.
- Because those guarded runs were executed under live workstation load, do not use them as apples-to-apples capacity comparisons between `stock` and `butterfly`.
- Use them as evidence that this workstation was not in a safe state for an honest `131072` or `262144` deployment claim without risking OOM.

Operational cap guidance

- Do not claim `262144` deployment-ready for Qwen 3.5 4B on this workstation.
- Keep the public Apple Silicon deployment cap conservative.
- Treat `131072` as something that still needs a dedicated quiet-host recheck under no-OOM conditions before broad deployment language.

Quality bar

The current Butterfly prefill path is enough for internal 4B deployment.

The current Butterfly prefill path is not enough for broader production rollout yet.

Minimum gate before broader rollout:

1. Long-context perplexity versus stock attention.
2. Needle-style long-context retrieval.
3. RULER-style long-context checks if available.
4. Small real-chat regression set at `4K`, `16K`, `64K`, and `128K`.

Open work

- keep validating that Butterfly prefill stays better than stock on Qwen 3.5 4B
